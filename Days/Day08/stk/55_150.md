# Evaluate Reverse Polish Notation

## Difficulty
Medium

## Description
Evaluate the value of an arithmetic expression in Reverse Polish Notation (postfix). Valid operators: +, -, *, /. Each operand may be an integer (positive/negative). Division between two integers should truncate toward zero.

## Examples

### Example 1

```text
Input: ["2","1","+","3","*"]
Output: 9
Explanation: (2 + 1) * 3 = 9
```

### Example 2

```text
Input: ["4","13","5","/","+"]
Output: 6
Explanation: 4 + (13 / 5) = 6 (truncate toward zero)
```

### Example 3

```text
Input: ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]
Output: 22
```

## Constraints

```text
1 <= tokens.length <= 10^4
tokens[i] is an operator: "+", "-", "*", "/" or an integer in string form
Integer values fit in 32-bit signed range
Division result must truncate toward zero
```

---

## Idea
Use a stack: push numbers, pop two on operator, compute a op b, push result.

---

## Pseudo Code

```text
function evaluateRPN(tokens)
    create empty stack S
    for each token in tokens
        if token is not an operator
            push integer(token) onto S
        else
            b = pop S
            a = pop S
            result = applyOperator(a, b, token)
            push result onto S
    return pop S

function applyOperator(a, b, op)
    if op == "+"
        return a + b
    if op == "-"
        return a - b
    if op == "*"
        return a * b
    if op == "/"
        return truncate_towards_zero_division(a, b)

function truncate_towards_zero_division(a, b)
    sign = 1 if (a >= 0) == (b >= 0) else -1
    absResult = floor(abs(a) / abs(b))
    return sign * absResult
```

## Code Explanation

### Dry Run

| Step | Token | Stack State       | Operation                 | Result |
| ---- | ----- | ----------------- | ------------------------- | ------ |
| 1    | "2"   | [2]               | push 2                    | [2]    |
| 2    | "1"   | [2,1]             | push 1                    | [2,1]  |
| 3    | "+"   | [2,1]             | pop 1, pop 2, 2+1 -> 3    | [3]    |
| 4    | "3"   | [3,3]             | push 3                    | [3,3]  |
| 5    | "*"   | [3,3]             | pop 3, pop 3, 3*3 -> 9    | [9]    |
| 6    | end   | [9]               | result                    | 9      |

## Other Patterns

### Pattern 1

**Idea**

Build expression tree from RPN and evaluate recursively.

**Pseudo Code**

```text
function buildTree(tokens)
    create empty stack S
    for each token in tokens
        if token is not an operator
            push new Node(value=integer(token)) onto S
        else
            right = pop S
            left = pop S
            push new Node(op=token, left=left, right=right) onto S
    return pop S

function evalTree(node)
    if node is leaf
        return node.value
    a = evalTree(node.left)
    b = evalTree(node.right)
    return applyOperator(a, b, node.op)
```

### Pattern 2

**Idea**

Recursive evaluation by scanning tokens from end using index pointer.

**Pseudo Code**

```text
function evalFromEnd(tokens)
    index = tokens.length - 1
    return helper(tokens, index).value

function helper(tokens, index)
    token = tokens[index]
    if token is not operator
        return (integer(token), index - 1)
    rightValue, index = helper(tokens, index - 1)
    leftValue, index = helper(tokens, index)
    return (applyOperator(leftValue, rightValue, token), index)
```

## Comparison

| Approach          | Idea                                 | Time   | Space  | Recommended |
| ----------------- | ------------------------------------ | ------ | ------ | ----------- |
| Stack (main)      | Push numbers, compute on operators   | O(n)   | O(n)   | Yes        |
