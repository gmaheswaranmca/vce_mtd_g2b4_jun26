# 012. LC #11 - Container With Most Water

| Field               | Value                                                                                                                                                                                                                                    |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **LC#**             | 11                                                                                                                                                                                                                                       |
| **Title**           | Container With Most Water                                                                                                                                                                                                                |
| **Difficulty**      | Medium                                                                                                                                                                                                                                   |
| **Description**     | Given an integer array `height` where `height[i]` represents the height of a vertical line at index `i`, find two lines that together with the x-axis form a container that stores the maximum amount of water. Return the maximum area. |
| **Idea (One Line)** | Start from both ends and always move the pointer with the smaller height.                                                                                                                                                                |

---

## Example 1
```text
Input: height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation:
Index : 0 1 2 3 4 5 6 7 8
Height: 1 8 6 2 5 4 8 3 7

Choose:

height[1] = 8
height[8] = 7

Width = 8 - 1 = 7

Area = min(8,7) × 7
     = 7 × 7
     = 49
```

---

## Example 2

```text
Input:height = [1,1]
Output:1
Explanation:
Width = 1

Height = min(1,1) = 1

Area = 1 × 1 = 1
```

---

## Constraints

```text
2 <= height.length <= 10^5

0 <= height[i] <= 10^4
```

---

# Main Pattern

## Two Pointers From Both Ends

### Idea (One Line)

Calculate area using left and right pointers, then move the shorter line inward.

---

# Important Formula

```text
Area = Width × Height
Width = right - left
Height = min(height[left], height[right])
Area = (right-left) × min(height[left],height[right])
```

---

# Why Move the Smaller Height?

Suppose

```text
left height = 3
right height = 8
```

Area is limited by:

```text
min(3,8) = 3
```

Moving the taller line:

```text
8 -> maybe 7 or 9
```

Still:

```text
min(3, anything) <= 3
```

And width decreases.

So area cannot improve.

Hence:

```text
Move the smaller height only.
```

This is the key observation.

---

# Pseudo Code

```text
left = 0
right = length(height)-1
maxArea = 0
while left < right
    width = right-left
    currArea = width * min(height[left],height[right])
    maxArea = max(maxArea,currArea)
    if height[left] < height[right]
        left = left + 1
    else
        right = right - 1
return maxArea
```

# Code Explanation (Simple for Students)

### Step 1
```
1. Place pointers:
    left  = 0
    right = n-1
2. Find area:
    area = (right-left) × min(height[left],height[right])
3. Update answer:
    maxArea = max(maxArea,area)
4.Move the smaller height.
If height[left] < height[right] move: 
    left++
otherwise:
    right--
5. Repeat until: left >= right
6. Return: maxArea
```

---

# Complexity

| Operation | Complexity |
| --------- | ---------: |
| Time      |       O(n) |
| Space     |       O(1) |

Because:

```text
Each pointer moves at most n times.
```

# Other Patterns

## Pattern 1 : Brute Force

### Idea (One Line)
Try every pair of lines and compute area.

### Pseudo Code
```text
maxArea = 0
for i = 0 to n-1
    for j = i+1 to n-1
        width = j-i
        area = width * min(height[i],height[j])
        maxArea = max(maxArea,area)
return maxArea
```

---

## Pattern 2 : Recursive Two Pointers
### Idea (One Line)
Use recursion instead of loop while shrinking the window.

### Pseudo Code
```text
function solve(left,right,maxArea)
    if left >= right
        return maxArea
    width = right-left
    area = width * min(height[left],height[right])
    maxArea = max(maxArea,area)
    if height[left] < height[right]
        return solve(left+1,right,maxArea)
    return solve(left,right-1,maxArea)
return solve(0,n-1,0)
```

---

## Pattern 3 : Monotonic Stack (Not Recommended)
### Idea (One Line)
Maintain candidate heights in stack and compute areas with stored indices.

### Pseudo Code
```text
create stack
for i from 0 to n-1
    while stack not empty
        calculate possible area
    push i
return maxArea
```

# Comparison of Approaches
| Pattern                |  Time |      Space | Recommended      |
| ---------------------- | ----: | ---------: | ---------------- |
| Brute Force            | O(n²) |       O(1) | ❌ No             |
| Two Pointers           |  O(n) |       O(1) | ✅ Best           |
| Recursive Two Pointers |  O(n) | O(n) stack | Learning         |
| Monotonic Stack        |  O(n) |       O(n) | Not natural here |

---

# Why Two Pointers is Optimal
Brute force:
```text
Check every pair (0,1) (0,2) (0,3)
...
Total O(n²)
```

Two pointers:
```text
Start widest container.
Discard impossible candidates intelligently.
Move smaller height only.
Each pointer moves once.
```

Hence:
```text
Time = O(n)
Space = O(1)
```

# Interview Takeaway
```text
Pattern : Two Pointers
Start: left = 0, right = n-1
Area: (right-left) × min(height[left],height[right])
Move: Smaller height inward
Answer: Maximum area found during traversal
```

This is one of the most famous **Two Pointer Opposite Ends** interview patterns and is frequently asked in coding interviews.
