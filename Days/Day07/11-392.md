# 011. LC #392 - Is Subsequence

| Field               | Value                                                                                                             |
| ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **LC#**             | 392                                                                                                               |
| **Title**           | Is Subsequence                                                                                                    |
| **Difficulty**      | Easy                                                                                                              |
| **Description**     | Given two strings `s` and `t`, return `true` if `s` is a subsequence of `t`, otherwise return `false`.            |
| **Subsequence**     | A subsequence is formed by deleting some characters (or none) without changing the order of remaining characters. |
| **Idea (One Line)** |      order.                                                 |

---

## Example 1
```text
Input: s = "abc", t = "ahbgdc"
Output: true
Explanation:
t = a h b g d c
    ↑   ↑     ↑
s = a b c
All characters of s are found in order.
```

---

## Example 2
```text
Input: s = "axc", t = "ahbgdc"
Output:false
Explanation:
Need a -> x -> c

a found
x not found afterwards

Hence false.
```

## Constraints

```text
0 <= s.length <= 100
0 <= t.length <= 10^4

s and t consist only of English letters.
```

---

# Main Pattern

## Two Pointers

### Idea
```text
Move:
* i -> s
* j -> t
If characters match: s[i] == t[j]
    move both.
Otherwise:
    move only `j`.
At the end:
    i == s.length
        means all characters are matched.
```
---

# Pseudo Code

```text
i = 0
j = 0
while i < length(s) and j < length(t)
    if s[i] == t[j]
        i = i + 1
    j = j + 1
return i == length(s)
```

# Code Explanation (Simple for Students)
```text
Suppose: s = abc, t = ahbgdc
We keep two fingers.
    i -> s
    j -> t
Start:
s : a b c
    ^
t : a h b g d c
    ^
Characters match: a == a
Move both.
Now
s : a b c
      ^
t : a h b g d c
      ^
b != h
    Skip `h`.
    Move only `j`.
Now
s : a b c
      ^
t : a h b g d c
        ^
b == b
    Move both.
c != g
c != d
    Skip them.
c == c
    Move both.
Now: i == length(s)
which means:
    Every character of s was found in order.
```

# Complexity

| Operation | Complexity |
| --------- | ---------: |
| Time      |       O(n) |
| Space     |       O(1) |

where

```text
n = length(t)
```

because `t` is scanned only once.

# Other Patterns

## Pattern 1 : Recursion

### Idea (One Line)

Match current characters and recursively move forward.

### Pseudo Code

```text
function solve(i,j)

if i == length(s)
    return true

if j == length(t)
    return false

if s[i] == t[j]
    return solve(i+1,j+1)

return solve(i,j+1)

return solve(0,0)
```

---

## Pattern 2 : Stack (Reverse Matching)

### Idea (One Line)

Push characters of s into stack from right and pop when matched in t.

### Pseudo Code

```text
create stack

for i from length(s)-1 downto 0
    push s[i]

for each ch in t

    if stack not empty and ch == top(stack)
        pop

return stack is empty
```

---

## Pattern 3 : Binary Search (Many Queries)

Useful when:

```text
One t
Many s queries
```

Example:

```text
t = "ahbgdc"

Check:

abc
adc
agc
axc
...
```

### Idea (One Line)

Store positions of characters in t and binary search next valid index.

### Pseudo Code

```text
store indices of each character in map

prev = -1

for each ch in s

    positions = map[ch]

    idx = first position > prev

    if idx not found
        return false

    prev = idx

return true
```

# Comparison of Approaches

| Pattern                 |       Time |      Space | Best Use                           |
| ----------------------- | ---------: | ---------: | ---------------------------------- |
| Two Pointers            |       O(n) |       O(1) | Single query                       |
| Recursion               |       O(n) | O(n) stack | Learning recursion                 |
| Stack                   |       O(n) |       O(m) | Alternative approach               |
| Binary Search + Indices | O(m log n) |       O(n) | Many subsequence queries           |
| DP                      |     O(m×n) |     O(m×n) | General sequence matching problems |

where

```text
m = length(s)
n = length(t)
```

## Interview Takeaway

For **LC 392**, the expected and most efficient solution is:

```text
Pattern = Two Pointers
i -> s
j -> t

Match -> move both
No match -> move t pointer only

If i reaches end of s
return true
```

This is one of the most important **Two Pointer (Different Length Strings)** patterns that frequently appears in interview problems.
