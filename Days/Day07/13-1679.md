# 013. LC #1679 - Max Number of K-Sum Pairs

| Field               | Value                                                                                                                                                                                      |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **LC#**             | 1679                                                                                                                                                                                       |
| **Title**           | Max Number of K-Sum Pairs                                                                                                                                                                  |
| **Difficulty**      | Medium                                                                                                                                                                                     |
| **Description**     | Given an integer array `nums` and an integer `k`, in one operation pick two numbers whose sum equals `k` and remove them from the array. Return the maximum number of operations possible. |
| **Idea (One Line)** | Sort the array and use two pointers from both ends to find pairs with sum `k`.                                                                                                             |

---

## Example 1

```text
Input: nums=[1,2,3,4], k=5
Output: 2
Explanation:
(1,4) -> sum=5
(2,3) -> sum=5
Maximum operations = 2
```

---

## Example 2

```text
Input: nums=[3,1,3,4,3], k=6
Output: 1
Explanation:
(3,3) -> sum=6
Remaining: [1,4,3]
No more valid pairs.
```

---

## Constraints

```text
1 <= nums.length <= 10^5
1 <= nums[i] <= 10^9
1 <= k <= 10^9
```

---

# Main Pattern

## Sort + Two Pointers

### Idea

```text
Sort the array.
If sum < k -> move left.
If sum > k -> move right.
If sum == k -> count pair and move both.
```

---

# Pseudo Code

```text
sort(nums)
left = 0
right = length(nums)-1
count = 0
while left < right
    sum = nums[left] + nums[right]
    if sum == k
        count = count + 1
        left = left + 1
        right = right - 1
    else if sum < k
        left = left + 1
    else
        right = right - 1
return count
```

---

# Code Explanation 
```text
1. Sort the array. 
nums=[3,1,3,4,3]
After sorting:
nums=[1,3,3,3,4]
1. Take two pointers.
    left=0
    right=n-1
2. Compute: sum = nums[left]+nums[right]
    1. If: sum==k
        We found one operation.
        count++
        left++
        right--
    2. If: sum<k
        Need larger sum.
        left++
    3. If:sum>k
        Need smaller sum.
        right--
3. Repeat until: left>=right
4. Return: count
```

# Other Patterns

## Pattern 1 : Hash Map Frequency (Optimal)
### Idea
```text
Store counts of numbers and search for k-num.
```

### Pseudo Code
```text
create map
count = 0
for each num in nums
    target = k-num
    if map[target] > 0
        count = count + 1
        map[target] = map[target]-1
    else
        map[num] = map[num]+1
return count
```

---

## Pattern 2 : Brute Force
### Idea
```text
Try every unused pair and count valid sums.
```
### Pseudo Code
```text
count = 0
mark all unused
for i=0 to n-1
    if used[i]
        continue
    for j=i+1 to n-1
        if used[j]
            continue
        if nums[i]+nums[j]==k
            used[i]=true
            used[j]=true
            count++
            break
return count
```

---

## Pattern 3 : Multiset / Balanced BST
### Idea
```text
Insert numbers into multiset and search k-num.
```
### Pseudo Code
```text
create multiset
count = 0
for each num
    target = k-num
    if target exists
        remove target
        count++
    else
        insert num
return count
```

# Comparison of Approaches

| Pattern             |       Time | Space | Recommended |
| ------------------- | ---------: | ----: | ----------- |
| Sort + Two Pointers | O(n log n) |  O(1) | ✅ Very Good |
| Hash Map Frequency  |       O(n) |  O(n) | ✅ Best      |
| Multiset            | O(n log n) |  O(n) | Possible    |
| Brute Force         |      O(n²) |  O(n) | ❌ TLE       |

---

# Which One is Preferred?

| Situation                   | Best Pattern        |
| --------------------------- | ------------------- |
| Array can be sorted         | Sort + Two Pointers |
| Need fastest time           | Hash Map            |
| Interview Two Pointer topic | Sort + Two Pointers |
| Very large input            | Hash Map O(n)       |

# Interview Takeaway

```text
Pattern: Sort + Two Pointers

Sort nums

left=0
right=n-1

sum=nums[left]+nums[right]

sum==k -> count++, left++, right--
sum<k -> left++
sum>k -> right--

Return count
```

This problem is a classic **Two Pointers After Sorting** pattern and is one of the most common interview problems involving pairs with a target sum.
