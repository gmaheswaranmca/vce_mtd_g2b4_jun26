# 018. Find Pivot Index

* **LC #:** 724
* **Difficulty:** Easy
* **Pattern:** Prefix Sum / Running Sum

## Title

**Find Pivot Index**

---

## Description

Given an integer array `nums`, calculate the pivot index of this array.

The pivot index is the index where:

* The sum of all the numbers strictly to the left of the index is equal to
* The sum of all the numbers strictly to the right of the index.

If the index is on the left edge, then the left sum is `0`.

If the index is on the right edge, then the right sum is `0`.

Return the leftmost pivot index. If no such index exists, return `-1`.

---

## Examples

```text
Input: nums=[1,7,3,6,5,6]
Output: 3

Explanation:

Index=3
Left sum = 1+7+3 = 11
Right sum = 5+6 = 11

Therefore pivot index = 3.
```

```text
Input: nums=[1,2,3]
Output: -1

Explanation:

Index 0: left=0,right=5
Index 1: left=1,right=3
Index 2: left=3,right=0

No pivot exists.
```

```text
Input: nums=[2,1,-1]
Output: 0

Explanation:

Index=0
Left sum = 0
Right sum = 1+(-1)=0

Pivot index = 0.
```

---

## Constraints

```text
1<=nums.length<=10^4
-1000<=nums[i]<=1000
```

---

# Idea (One Line)

Find total sum once, then for each index compute right sum as `total-left-nums[i]`.

---

# Pseudocode

```text
function pivotIndex(nums)
    total=0
    for each num in nums
        total=total+num
    left=0
    for i=0 to length(nums)-1
        right=total-left-nums[i]
        if left==right
            return i
        left=left+nums[i]
    return -1
```

---

# Complexity

```text
Time Complexity: O(n)
One pass for total sum.
One pass to find pivot.
Space Complexity: O(1)
```

# Code Explanation

```text
1. Calculate total sum of the array.
2. Initialize left=0.
3. Traverse array from left to right.
4. For each index:
   right=total-left-nums[i]
5. Compare:
   if left==right
   return i
6. Update:
   left=left+nums[i]
7. If traversal finishes without match,
   return -1.
```

---

# Why right=total-left-nums[i] ?
Suppose:
```text
nums=[1,7,3,6,5,6]
total=28
At i=3
left=11
right=28-11-6
     =11
```
Because:
```text
total=left+nums[i]+right
Therefore,
right=total-left-nums[i]
```
This avoids computing right sum repeatedly.

# Other Patterns

## Pattern 1 : Prefix Sum Array

### Idea

Store prefix sums and compute left/right sums using prefix values.

### Pseudocode

```text
function pivotIndex(nums)
    n=length(nums)
    prefix[0]=nums[0]
    for i=1 to n-1
        prefix[i]=prefix[i-1]+nums[i]
    total=prefix[n-1]
    for i=0 to n-1
        if i==0
            left=0
        else
            left=prefix[i-1]
        right=total-prefix[i]
        if left==right
            return i
    return -1
```

### Complexity

```text
Time: O(n)
Space: O(n)
```

---

## Pattern 2 : Brute Force

### Idea

For every index, calculate left sum and right sum separately.

### Pseudocode

```text
function pivotIndex(nums)
    n=length(nums)
    for i=0 to n-1
        left=0
        for j=0 to i-1
            left=left+nums[j]
        right=0
        for j=i+1 to n-1
            right=right+nums[j]
        if left==right
            return i
    return -1
```

### Complexity

```text
Time: O(n^2)
Space: O(1)
```

# Key Observation

```text
Instead of:
    Computing left sum every time -> O(n)
    Computing right sum every time -> O(n)
Keep:
    left as running sum
    right=total-left-nums[i]
Thus:
    Overall complexity becomes O(n).
```
