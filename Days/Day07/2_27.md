# Remove Element

## Difficulty
Easy

## Description
Remove all occurrences of val from array nums in-place and return the new length k. The relative order of elements may be changed. Do not allocate extra space for another array; perform the operation with O(1) extra memory.

## Examples

### Example 1

```text
Input: nums = [3,2,2,3], val = 3
Output: 2, nums = [2,2,_,_]
Explanation: Return k = 2, the first two elements of nums are 2.
```

### Example 2

```text
Input: nums = [0,1,2,2,3,0,4,2], val = 2
Output: 5, nums = [0,1,3,0,4,_,_,_]
Explanation: Return k = 5. The first five elements contain the final result.
```

## Constraints

```text
0 <= nums.length <= 100000
-10^9 <= nums[i], val <= 10^9
Modify nums in-place with O(1) extra memory
```

---

## Idea
Two-pointer (fast-slow) overwrite unwanted values: copy non-val elements to front using a write pointer.

---

## Pseudo Code

```text
function removeElement(nums, val)
    write = 0
    for read from 0 to length(nums)-1
        if nums[read] != val
            nums[write] = nums[read]
            write = write + 1
    return write
```

---

## Complexity

| Metric           | Value  |
| ---------------- | ------ |
| Time Complexity  | O(n)   |
| Space Complexity | O(1)   |

---

## Code Explanation

### Dry Run

| Step | read | write | nums state             | Operation                     | Result         |
| ---- | ---- | ----- | ---------------------- | ----------------------------- | -------------- |
| 1    | 0    | 0     | [3,2,2,3]              | nums[0]==3 == val -> skip     | write=0        |
| 2    | 1    | 0     | [3,2,2,3]              | nums[1]==2 != val -> copy     | nums->[2,2,2,3], write=1 |
| 3    | 2    | 1     | [2,2,2,3]              | nums[2]==2 != val -> copy     | nums->[2,2,2,3], write=2 |
| 4    | 3    | 2     | [2,2,2,3]              | nums[3]==3 == val -> skip     | write=2, return 2 |

1. Initialize write pointer at start.
2. Iterate read pointer over array.
3. When element != val, write it at write and increment write.
4. Return write as new length.

## Other Patterns

### Pattern 2

**Idea**

Two-pointer swap with end when order doesn't matter.

**Pseudo Code**

```text
function removeElementSwap(nums, val)
    n = length(nums)
    i = 0
    while i < n
        if nums[i] == val
            nums[i] = nums[n-1]
            n = n - 1
        else
            i = i + 1
    return n
```

### Pattern 3

**Idea**

Create a new array (uses extra space).

**Pseudo Code**

```text
function removeElementExtra(nums, val)
    out = empty list
    for x in nums
        if x != val
            append x to out
    for i from 0 to length(out)-1
        nums[i] = out[i]
    return length(out)
```

### Pattern 4

**Idea**

Repeated removal using built-in remove (inefficient for arrays).

**Pseudo Code**

```text
function removeElementRepeated(nums, val)
    i = 0
    while i < length(nums)
        if nums[i] == val
            remove element at index i from nums
        else
            i = i + 1
    return length(nums)
```

## Comparison

| Approach                | Idea                             | Time    | Space | Recommended |
| ----------------------- | -------------------------------- | ------- | ----- | ----------- |
| Two-pointer overwrite   | Copy non-val elements forward    | O(n)    | O(1)  | Yes         |
| Swap with end           | Replace val with last element    | O(n)    | O(1)  | Yes (if order not required) |
| Extra array             | Build new array of kept elements | O(n)    | O(n)  | No          |
| Repeated remove         | Remove in-place with shifts      | O(n^2)  | O(1)  | No          |
