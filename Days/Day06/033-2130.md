## 033. Maximum Twin Sum of a Linked List

| Item        | Value                                                    |
| ----------- | -------------------------------------------------------- |
| LC #        | 2130                                                     |
| Difficulty  | Medium                                                   |
| Title       | Maximum Twin Sum of a Linked List                        |
| Description | Find the maximum twin sum of an even-length linked list. |
| Pattern     | Fast & Slow Pointer + Reverse Linked List                |
| Idea        | Find middle, reverse second half, compare twin pairs.    |

---

## Description

In a linked list of size `n`, where `n` is even:

* Node `0` is the twin of node `n-1`
* Node `1` is the twin of node `n-2`
* Node `2` is the twin of node `n-3`
* and so on.

The twin sum of a node is:

```text
node[i].val+node[n-1-i].val
```

Return the maximum twin sum of the linked list.

---

## Examples

```text
Input: head=[5,4,2,1]
Output: 6

Explanation:
Twin pairs:
(5,1)=6
(4,2)=6

Maximum twin sum=6
```

```text
Input: head=[4,2,2,3]
Output: 7

Explanation:
Twin pairs:
(4,3)=7
(2,2)=4

Maximum twin sum=7
```

```text
Input: head=[1,100000]
Output: 100001

Explanation:
Twin pair:
(1,100000)=100001
```

---

## Constraints

```text
2<=Number of nodes<=10^5
Number of nodes is even
1<=Node.val<=10^5
```

---

## Idea (One Line)

Find the middle using fast and slow pointers, reverse the second half, then compute maximum pair sums.

---

## Pseudocode

```text
function pairSum(head)

    slow=head
    fast=head

    while fast!=null and fast.next!=null

        slow=slow.next
        fast=fast.next.next

    prev=null
    current=slow

    while current!=null

        nextNode=current.next

        current.next=prev

        prev=current

        current=nextNode

    first=head
    second=prev

    maxSum=0

    while second!=null

        maxSum=max(maxSum,first.val+second.val)

        first=first.next

        second=second.next

    return maxSum
```

---

## Dry Run

### Input

```text
head=[5,4,2,1]

5->4->2->1
```

### Step 1 : Find Middle

| Iteration | slow | fast |
| --------- | ---: | ---: |
| Initial   |    5 |    5 |
| 1         |    4 |    2 |
| 2         |    2 | null |

```text
slow points to node 2
Second half starts at 2->1
```

---

### Step 2 : Reverse Second Half

Original:

```text
2->1->null
```

After reversal:

```text
1->2->null
```

| prev | current | nextNode |
| ---- | ------- | -------- |
| null | 2       | 1        |
| 2    | 1       | null     |
| 1->2 | null    | -        |

---

### Step 3 : Compute Twin Sums

First Half:

```text
5->4
```

Reversed Second Half:

```text
1->2
```

| first | second | Twin Sum | maxSum |
| ----- | -----: | -------: | -----: |
| 5     |      1 |        6 |      6 |
| 4     |      2 |        6 |      6 |

Return:

```text
6
```

---

## Code Explanation

```text
1. Use slow and fast pointers to locate the middle.
2. slow moves one step and fast moves two steps.
3. When fast reaches end, slow reaches second half.
4. Reverse the second half of the linked list.
5. first starts from head.
6. second starts from reversed second half.
7. Calculate first.val+second.val.
8. Update maxSum.
9. Move both pointers forward.
10. Return maxSum.
```

```text
Time Complexity: O(n)
```

# Other Patterns

## Pattern 1: Array + Two Pointers

### Idea

Store all node values in an array and use two pointers from both ends.

### Pseudocode

```text
function pairSum(head)

    nums=emptyList()

    current=head

    while current!=null

        add current.val to nums

        current=current.next

    left=0

    right=length(nums)-1

    answer=0

    while left<right

        answer=max(answer,nums[left]+nums[right])

        left=left+1

        right=right-1

    return answer
```

---

## Pattern 2: Stack + Middle Pointer

### Idea

Push first half into stack and compare with second half.

### Pseudocode

```text
function pairSum(head)

    slow=head

    fast=head

    stack=emptyStack()

    while fast!=null and fast.next!=null

        push slow.val into stack

        slow=slow.next

        fast=fast.next.next

    answer=0

    while slow!=null

        answer=max(answer,pop stack+slow.val)

        slow=slow.next

    return answer
```

---

## Pattern 3: Recursion

### Idea

Use recursion with front pointer and compare symmetric nodes.

### Pseudocode

```text
function pairSum(head)

    front=head

    answer=0

    recursive(head)

    return answer


function recursive(node)

    if node==null

        return

    recursive(node.next)

    answer=max(answer,front.val+node.val)

    front=front.next
```

---

## Pattern Comparison

| Pattern               | Idea                                | Time | Space | Recommended |
| --------------------- | ----------------------------------- | ---: | ----: | ----------- |
| Fast & Slow + Reverse | Reverse second half and compare     | O(n) |  O(1) | ⭐⭐⭐⭐⭐ Best  |
| Array + Two Pointers  | Store values then compare ends      | O(n) |  O(n) | ⭐⭐⭐⭐        |
| Stack + Middle        | Push first half to stack            | O(n) |  O(n) | ⭐⭐⭐         |
| Recursion             | Compare symmetric nodes recursively | O(n) |  O(n) | ⭐⭐          |

## Important Observation

```text
Twin indices are:

0<->n-1
1<->n-2
2<->n-3
...
i<->n-1-i

Example:

Index: 0 1 2 3
Value: 5 4 2 1

Twin Pairs:

5+1=6
4+2=6

Maximum=6
```

```text
This problem combines three famous Linked List patterns:

1. Fast & Slow Pointer (Find Middle)
2. Reverse Linked List
3. Two Pointer Traversal

It is one of the most important medium-level linked list interview problems.
```
