## 031. Odd Even Linked List

| Item        | Value                                                                                                                                                                                    |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| LC #        | 328                                                                                                                                                                                      |
| Difficulty  | Medium                                                                                                                                                                                   |
| Title       | Odd Even Linked List                                                                                                                                                                     |
| Description | Rearrange the linked list such that all nodes at odd indices are grouped together followed by nodes at even indices. The relative order inside odd and even groups must remain the same. |
| Pattern     | Linked List Pointer Manipulation                                                                                                                                                         |
| Idea        | Maintain odd and even chains separately                                                                                                                                                  |

---

### Description
```text
Given the head of a singly linked list, 
group all nodes with odd indices together 
followed by nodes with even indices.
    * The first node is considered odd.
    * The second node is considered even.
    * Relative order inside odd and even groups should remain unchanged.
    * Solve in O(1) extra space and O(n) time.
```

---

### Examples

```text
Input: head=[1,2,3,4,5]
Output: [1,3,5,2,4]
Explanation:
Odd indexed nodes: 1->3->5
Even indexed nodes: 2->4
Join odd list with even list.
```

```text
Input: head=[2,1,3,5,6,4,7]
Output: [2,3,6,7,1,5,4]
Explanation:
Odd indexed nodes: 2->3->6->7
Even indexed nodes: 1->5->4
Append even chain after odd chain.
```

---

### Constraints

```text
0<=Number of nodes<=10^4
-10^6<=Node.val<=10^6
```

---

## Idea (One Line)

Separate odd and even indexed nodes into two chains and connect them at the end.

---

## Pseudocode

```text
function oddEvenList(head)
    if head == null or head.next == null
        return head
    odd = head
    even = head.next
    evenHead = even
    while even != null and even.next != null
        odd.next = even.next
        odd = odd.next

        even.next = odd.next
        even = even.next
    odd.next = evenHead
    return head
```

---

## Dry Run
```text
Input: 1->2->3->4->5
Initial:
    odd=1
    even=2
    evenHead=2
List: 1->2->3->4->5
```
| Step    | odd | even | Operation         | List State |
| ------- | --: | ---: | ----------------- | ---------- |
| Initial |   1 |    2 | Setup pointers    | 1→2→3→4→5  |
| 1       |   1 |    2 | odd.next=3        | 1→3        |
| 2       |   3 |    2 | odd=odd.next      | odd=3      |
| 3       |   3 |    2 | even.next=4       | 2→4        |
| 4       |   3 |    4 | even=even.next    | even=4     |
| 5       |   3 |    4 | odd.next=5        | 1→3→5      |
| 6       |   5 |    4 | odd=odd.next      | odd=5      |
| 7       |   5 |    4 | even.next=null    | 2→4        |
| 8       |   5 | null | even=even.next    | even=null  |
| Final   |   5 | null | odd.next=evenHead | 1→3→5→2→4  |

---

### Code Explanation
```text
1. Handle empty list or single node list.
2. odd points to first node.
3. even points to second node.
4. Store evenHead to reconnect later.
5. Move odd.next to next odd node.
6. Advance odd pointer.
7. Move even.next to next even node.
8. Advance even pointer.
9. Continue until no more even nodes exist.
10. Attach even chain after odd chain.
11. Return original head.
```

---

## Complexity
```text
Time Complexity: O(n)
Space Complexity: O(1)
```

# Other Patterns
## Pattern 1: Two Separate Dummy Lists
### Idea
Create odd and even lists separately using dummy nodes, then join them.

### Pseudocode
```text
function oddEvenList(head)
    oddDummy=new Node(0)
    evenDummy=new Node(0)

    oddTail=oddDummy
    evenTail=evenDummy

    index=1
    current=head
    while current!=null
        nextNode=current.next
        current.next=null
        if index%2==1
            oddTail.next=current
            oddTail=current
        else
            evenTail.next=current
            evenTail=current
        current=nextNode
        index=index+1
    oddTail.next=evenDummy.next
    return oddDummy.next
```

---

## Pattern 2: Array of Nodes
### Idea
Store odd nodes and even nodes separately in arrays and reconnect.
### Pseudocode
```text
function oddEvenList(head)
    oddNodes=emptyList()
    evenNodes=emptyList()

    current=head
    index=1
    while current!=null
        if index%2==1
            add current to oddNodes
        else
            add current to evenNodes
        current=current.next
        index=index+1
    connect all nodes in oddNodes
    connect all nodes in evenNodes
    connect last odd node to first even node
    return first odd node
```

## Pattern Comparison
| Pattern                    | Idea                            | Time | Space | Recommended |
| -------------------------- | ------------------------------- | ---: | ----: | ----------- |
| In-place Odd/Even Pointers | Rearrange pointers directly     | O(n) |  O(1) | ⭐⭐⭐⭐⭐ Best  |
| Dummy Odd & Even Lists     | Build two lists then join       | O(n) |  O(1) | ⭐⭐⭐⭐        |
| Arrays of Nodes            | Store odd/even nodes separately | O(n) |  O(n) | ⭐⭐          |
| Queue Based                | Push odd/even nodes into queues | O(n) |  O(n) | ⭐           |

## Important Observation
```text
The problem groups nodes based on INDEX, not VALUE.
For:1->2->3->4->5
Odd indices : 1st,3rd,5th => 1->3->5
Even indices: 2nd,4th => 2->4
Result:1->3->5->2->4
```

**The in-place odd/even pointer approach is the standard and optimal solution expected in interviews and LeetCode.**
