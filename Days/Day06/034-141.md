## 034. Linked List Cycle

**Difficulty:** Easy

### Description

Given `head`, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the `next` pointer.

Return `true` if there is a cycle in the linked list. Otherwise, return `false`.

---

### Examples

```text
Input: head=[3,2,0,-4], pos=1
Output: true
Explanation:
Tail connects to node at index 1.
The linked list forms a cycle.
```

```text
Input: head=[1,2], pos=0
Output: true
Explanation:
Tail connects to first node.
The linked list forms a cycle.
```

```text
Input: head=[1], pos=-1
Output: false
Explanation:
There is no cycle in the linked list.
```

---

### Constraints

```text
0<=Number of nodes<=10^4
-10^5<=Node.val<=10^5
pos is -1 or a valid index in the linked list.
```

---

## Idea (One Line)

Use two pointers moving at different speeds; if they meet, a cycle exists.

---

## Pseudocode (Floyd Cycle Detection)

```text
function hasCycle(head)
    slow=head
    fast=head

    while fast!=null and fast.next!=null
        slow=slow.next
        fast=fast.next.next

        if slow==fast
            return true

    return false
```

---

## Complexity

```text
Time Complexity: O(n)
Space Complexity: O(1)
```

---

## Dry Run

### Input

```text
head=[3,2,0,-4], pos=1
```

Structure:

```text
3 -> 2 -> 0 -> -4
     ^         |
     |_________|
```

| Iteration | slow | fast | slow moves | fast moves | Meet? |
| --------- | ---: | ---: | ---------- | ---------- | ----- |
| 0         |    3 |    3 | -          | -          | No    |
| 1         |    2 |    0 | 3→2        | 3→0        | No    |
| 2         |    0 |    2 | 2→0        | 0→2        | No    |
| 3         |   -4 |   -4 | 0→-4       | 2→-4       | Yes   |

```text
slow==fast
Return true
```

---

## Code Explanation

```text
1. Initialize two pointers slow and fast at head.
2. Move slow by one step each iteration.
3. Move fast by two steps each iteration.
4. If slow and fast meet, a cycle exists.
5. If fast reaches null, no cycle exists.
6. Return false if loop ends.
```

---

# Why Floyd's Algorithm Works

```text
Suppose:

Distance before cycle=a
Cycle length=b

Slow travels=a+x
Fast travels=a+x+k*b

Since fast moves twice as fast:

2(a+x)=a+x+k*b

a+x=k*b

Therefore they eventually meet inside the cycle.
```

# Other Patterns

## 1. HashSet Pattern

### Idea

Store visited nodes in a set. If a node appears again, cycle exists.

### Pseudocode

```text
function hasCycle(head)
    visited=emptySet()

    current=head

    while current!=null
        if current in visited
            return true

        add current into visited
        current=current.next

    return false
```

---

## 2. Mark Visited Node Pattern

### Idea

Modify node pointers or values to mark visited nodes.

### Pseudocode

```text
function hasCycle(head)
    marker=new Node()

    current=head

    while current!=null
        if current.next==marker
            return true

        nextNode=current.next
        current.next=marker
        current=nextNode

    return false
```

---

## 3. Count Traversal With Limit Pattern

### Idea

Traverse nodes and stop if visited count exceeds total allowed nodes.

### Pseudocode

```text
function hasCycle(head)
    count=0
    current=head

    while current!=null
        count=count+1

        if count>10000
            return true

        current=current.next

    return false
```

# Comparison of Approaches

| Approach                | Idea                                  | Time | Space | Modify List | Recommended |
| ----------------------- | ------------------------------------- | ---: | ----: | ----------: | ----------: |
| Floyd Slow-Fast Pointer | Two pointers move at different speeds | O(n) |  O(1) |          No |       ⭐⭐⭐⭐⭐ |
| HashSet                 | Store visited nodes                   | O(n) |  O(n) |          No |        ⭐⭐⭐⭐ |
| Mark Visited            | Alter next pointer                    | O(n) |  O(1) |         Yes |          ⭐⭐ |
| Traversal Limit         | Count nodes and use bound             | O(n) |  O(1) |          No |           ⭐ |

## Best Choice

```text
Interview: Floyd Slow-Fast Pointer
Reason:
1. O(n) time.
2. O(1) extra space.
3. Does not modify linked list.
4. Standard interview solution for LC 141.
```
